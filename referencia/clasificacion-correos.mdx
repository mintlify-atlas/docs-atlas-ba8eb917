El archivo `docs/auxiliares/tags_project.csv` define un sistema de etiquetas para clasificar los correos de Gmail del proyecto y mapearlos a las categorías del diario en el grafo Logseq. Esta clasificación garantiza trazabilidad completa entre las comunicaciones externas y los registros internos del proyecto CALAIRE-EA, facilitando el seguimiento de decisiones, entregables y operaciones técnicas.

## Propósito del sistema

Cada correo de Gmail relevante para CALAIRE-EA recibe una o más etiquetas del sistema definido en `tags_project.csv`. La columna `Categoria_Journal` de ese CSV indica a cuál de las cinco categorías del diario de Logseq corresponde cada etiqueta, permitiendo un flujo directo: **correo Gmail → etiqueta → categoría del journal → bloque de evento**.

<Info>
La regla de mapeo es **uno a uno**: cada etiqueta corresponde exactamente a una sola categoría del journal. Esto evita ambigüedad al momento de registrar eventos en el diario.
</Info>

## Columna Categoria_Journal

La columna `Categoria_Journal` es el puente entre el correo y el grafo. Sus valores posibles son las cinco categorías estándar del journal:

| Valor en Categoria_Journal | Categoría en el diario |
|---|---|
| `[[Prueba Piloto]]` | Rondas EA, confirmaciones, logística |
| `[[Gestión Administrativa]]` | Comunicaciones, contratos, finanzas |
| `[[Desarrollo Técnico]]` | CALAIRE-APP, calibración, documentación técnica |
| `[[Infraestructura]]` | TI, instalaciones, transporte de equipos |
| `[[SGC / Calidad]]` | Auditorías, procedimientos ISO, acreditación |

## Flujo de clasificación en Gmail

El proceso de clasificación conecta el correo entrante con el registro en el grafo Logseq en tres pasos:

<Steps>
  <Step title="Recibir el correo en Gmail">
    Identificar el correo como relevante para CALAIRE-EA. Revisar remitente, asunto y contenido para determinar el grupo temático.
  </Step>
  <Step title="Aplicar la etiqueta correspondiente">
    Asignar la etiqueta del grupo que mejor describa el contenido del correo. Para mayor granularidad, aplicar también el subtag correspondiente. Consultar `docs/auxiliares/tags_project.csv` para verificar el mapeo.
  </Step>
  <Step title="Registrar en el grafo Logseq">
    Con base en la columna `Categoria_Journal` de la etiqueta aplicada, crear o actualizar el bloque de evento en la sección temática correspondiente del diario del día.
  </Step>
</Steps>

<Note>
La regla uno a uno garantiza que no exista ambigüedad: si un correo tiene la etiqueta `[TECH] Ronda - Equipos`, el evento siempre se registra bajo la categoría `[[Infraestructura]]` en el diario, no en otra sección.
</Note>
